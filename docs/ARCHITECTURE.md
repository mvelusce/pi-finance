## Data Flow

1. **User enters formula** in Google Sheets: `=STOCKPRICE("AAPL")`
2. **Apps Script** executes and makes HTTPS request to your domain
3. **Synology Reverse Proxy** receives HTTPS request, terminates SSL
4. **Forwards to Raspberry Pi** via local HTTP
5. **Docker Container** receives request on port 8000
6. **FastAPI validates** API key from `X-API-Key` header
7. **Calls yfinance** to fetch data from Yahoo Finance
8. **Returns JSON** response back through the chain
9. **Google Sheets** displays the result in the cell

## Scaling Considerations

### Current Setup (Single Pi)
- âœ… Perfect for personal use
- âœ… Low power consumption
- âœ… Minimal maintenance
- âš ï¸  Limited concurrent requests
- âš ï¸  Single point of failure

### Future Enhancements
1. **Add Redis Caching**
   - Cache stock quotes for 1-5 minutes
   - Reduce API calls to Yahoo Finance
   - Faster responses

2. **Load Balancing**
   - Multiple Raspberry Pis
   - Distribute requests
   - High availability

3. **Database**
   - Store historical data
   - Faster queries
   - Offline capability

4. **Rate Limiting**
   - Protect against abuse
   - Per-key limits
   - Usage monitoring

5. **Monitoring**
   - Prometheus + Grafana
   - Alert on failures
   - Performance metrics

## Why This Architecture?

### âœ… Advantages
1. **Simple** - Easy to understand and maintain
2. **Secure** - Multiple security layers
3. **Cost-effective** - Uses existing hardware (Pi + NAS)
4. **Reliable** - Minimal dependencies
5. **Flexible** - Easy to extend with new endpoints
6. **Documented** - Automatic API docs via FastAPI
7. **Portable** - Docker ensures consistent deployment
8. **Automated** - GitHub Actions for CI/CD

### ğŸ¯ Design Decisions

**Why FastAPI?**
- Fast, modern Python framework
- Automatic API documentation
- Type safety with Pydantic
- Async support for scalability

**Why Docker?**
- Consistent environment
- Easy updates
- Multi-architecture support
- Isolated from host system

**Why API Key Auth?**
- Simple to implement
- Easy to use from Apps Script
- No OAuth complexity
- Sufficient for private API

**Why Raspberry Pi?**
- Low power (< 5W)
- Always-on home server
- Sufficient performance
- Cost-effective

**Why Synology Reverse Proxy?**
- Already have the hardware
- Handles SSL/TLS
- Domain management
- No need for separate proxy server

---

This architecture balances simplicity, security, and functionality for personal finance tracking from Google Sheets.
